## GECOM41 — Reception Order Confirmation

**Reception order notification** message sent by the Generix Spain WMS to the source system (ERP). It reports the closing of the reception and the goods actually received by the warehouse for the order in question.

### Key Features

- **Flow direction:** Outbound — the WMS publishes this message to the client endpoint (`POST https://[URL_Client]/endpoint`). The ERP must expose and process it.
- **Identification:** The `REFREC` reference, together with the order number (`NUMREC`) and the supplier (`CODFOU`), links the confirmation with the original order. The order origin is indicated in `ORICDE` (interface, local entry, global reintegration or reception to be verified).
- **Tracking dates:** Distinguishes between the expected date (`DTIREC`), rescheduled date (`DTMREC`) and actual reception date (`DTRREC`), allowing the ERP to measure deviations.
- **Additional header information (GEEX4101):** Delivery note data, truck temperature, veterinary and customs information, and reception or rejection reasons.
- **Received lines (GEEX4120):** Array of up to 999 lines with the breakdown by product: quantity ordered (`UVCREA`), received (`UVCREC`), free (`UVCGRA`), immobilized (`UVCIMM`), rejected (`UVCRFU`) and on backorder/reliquat (`UVCRLQ`), with their associated reasons.
- **Detail by support (GEEX4130):** Nested in each line, it identifies batch, pallet (EAN128), storage location and serial numbers (`GEEX4140`).
- **Received packaging (GEEX4180):** Array of up to 99 entries for the management of returnable packaging (inbound/outbound).
- **Totals (GEEX4199):** Summary with the number of lines, supports, packages and products of the order.

> **Important note:** This message is generated by the WMS; the process code `TRTEXC` only accepts empty or `2` (modification). The ERP must use the `REFREC` reference to reconcile the confirmation with the corresponding **GECOM40** order.
>!green The **RCTEXC** field must be correctly populated, as it is essential for routing messages to the appropriate WMS library. The value for this field differs between the test and production environments. The correct value can be provided by the WMS consultant.

### Integration

This message is sent automatically by the WMS when the warehouse closes the reception of the order. The ERP must keep an active and accessible endpoint to receive the notifications and update the order information. The references `REFREC`, `CODACT`, `CODTRE`, `NLIREC`, `CODPRO`, etc. correspond to the same references declared in the **GECOM40** message.
