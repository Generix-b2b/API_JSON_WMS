## GECOM51 — Delivery Order Confirmation

**Delivery confirmation** message sent by the Generix Spain WMS to the source system (ERP).
It reports the actual result of a delivery order once it has been picked and dispatched: quantities served, prepared supports and transport data.

### Key Features

- **Flow direction:** Outbound — the WMS publishes this message to the client endpoint (`POST https://[URL_Client]/endpoint`). It is the response to the delivery order previously declared via **GECOM50**.
- **Identification:** The `REFLIV` reference, together with the order number (`NUMLIV`) and the customer (`CODCLI`), links the confirmation with the original order. The origin is indicated in `ORICDE` (GECOM interface or local entry).
- **Delivery settlement (LIVSOL):** Indicates whether the delivery is settled (`true`) or whether it generates a reliquat (`false`). The `INDRLQ` indicator flags the presence of a generated backorder.
- **Additional header information (GEEX5101):** Sea transport data (ship, shipping line, container), freight forwarder, shipment number and DAE references.
- **VPC delivery address (GEEX5110):** Optional block with the final recipient's address for e-commerce deliveries.
- **Delivered lines (GEEX5120):** Array of up to 999 lines with quantity ordered (`UVCCDE`), delivered (`UVCLIV`), served in the wave (`UVCSRV`) and initially ordered (`UVCINI`), plus discrepancy reasons.
- **Detail by support (GEEX5130):** Nested in each line, it describes pallet, batch, preparation type (picking), dangerous goods data, temperature and serial numbers (`GEEX5140`).
- **Totals (GEEX5199):** Summary with the number of lines, prepared supports, packages, products and gross and net weights of the order.

> **Important note:** This message is generated by the WMS; the process code `TRTEXC` always comes empty. The ERP must use the `REFLIV` reference to reconcile with the corresponding GECOM50 order.
>!green The **RCTEXC** field must be correctly populated, as it is essential for routing messages to the appropriate WMS library. The value for this field differs between the test and production environments. The correct value can be provided by the WMS consultant.

### Integration

The ERP must keep an active endpoint to receive these confirmations and update the status of its sales orders.
The quantities served and the support and transport data allow the ERP to issue the invoice, generate the DESADV to the final customer and close the order.
