## GECOM80 — Stock Movement

**Stock movement notification** message sent by the Generix Spain WMS to the source system (ERP).
It reports every physical or administrative movement that affects the warehouse inventory, allowing the ERP to keep its stock synchronized in real time.

### Key Features

- **Flow direction:** Outbound — the WMS publishes this message to the client endpoint (`POST https://[URL_Client]/endpoint`) each time a movement occurs.
- **Movement type (CODMVT):** Administrative (`00`), inbound/storage (`10`), outbound/destocking (`20`), transfer (`30`), quantity adjustment (`40`) or immobilization (`80`).
- **Movement direction (SENMVT):** `+` for storage or de-immobilization, `-` for outbound or immobilization.
- **Goods identification:** Product (`CODPRO`), logistics variant, batch (`CODLOT`), EAN128 pallet (`CODPAL`) and full location (zone, aisle, column, level).
- **Third party and associated order:** Third-party code (`CODTIE`) with its type (supplier/customer) and the reception or delivery order number (`NUMCDE`) that originated the movement.
- **Additional information (GEEX8001):** Analysis data, weights, SAP codes (reason, location, status) and product family.
- **Immobilization (GEEX8060):** Block for immobilization/de-immobilization movements, with dates, times and reasons.
- **Positive release (GEEX8090):** Block for complementary pallet-release movements, with pallet status (`ETAPAL`) and blacklist management (black list).

> **Important note:** This message is generated by the WMS; the process code `TRTEXC` always comes empty. The minimum required fields are the activity (`CODACT`), the product (`CODPRO`) and the movement type (`CODMVT`).
>!green The **RCTEXC** field must be correctly populated, as it is essential for routing messages to the appropriate WMS library. The value for this field differs between the test and production environments. The correct value can be provided by the WMS consultant.

### Integration

The ERP must keep an active endpoint to receive the movement notifications and update its stock accounting.
This message provides the detailed trace of each operation; for a complete snapshot of the inventory at a given moment, use **GECOM90**.
